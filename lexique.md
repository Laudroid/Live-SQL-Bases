**API** : Interface permettant à différents logiciels de communiquer entre eux.

**AVG** : Fonction d'agrégation qui calcule la moyenne des valeurs d'une colonne numérique.

**AND** : Opérateur logique qui combine deux conditions, les deux devant être vraies.

**Back-end** : Partie d'une application qui gère la logique métier, la base de données et la communication avec les serveurs.

**Bases de données relationnelles** : Système de gestion de données où les informations sont organisées en tables reliées par des clés.

**B-tree** : Type d'index couramment utilisé pour l'organisation hiérarchique des données, optimisé pour les recherches et les tris.

**Clés étrangères** : Colonne ou ensemble de colonnes dans une table qui fait référence à une clé primaire d'une autre table, établissant une relation.

**Clés primaires** : Colonne ou ensemble de colonnes qui identifient de manière unique chaque ligne d'une table.

**Colonnes** : Attributs ou champs d'une table, définissant le type de données stockées pour chaque ligne.

**Contraintes d'intégrité référentielle** : Règles assurant la cohérence des relations entre les tables en empêchant les actions qui briseraient ces liens.

**COUNT** : Fonction d'agrégation qui compte le nombre de lignes ou de valeurs non nulles dans une colonne.

**DELETE** : Commande SQL utilisée pour supprimer des lignes de données d'une table.

**Données relationnelles** : Données organisées selon le modèle relationnel, où elles sont stockées dans des tables liées entre elles.

**Expressions régulières** : Séquences de caractères qui définissent un motif de recherche, utilisées pour la correspondance de chaînes de caractères complexes.

**Fonctions d'agrégation** : Fonctions SQL qui effectuent un calcul sur un ensemble de lignes et renvoient une seule valeur.

**FULL JOIN** : Type de jointure qui renvoie toutes les lignes lorsque qu'il y a une correspondance dans l'une des tables.

**Full-Text Search** : Module de PostgreSQL offrant des capacités de recherche Full-Text avancées.

**Hash** : Type d'index basé sur une fonction de hachage, idéal pour les recherches d'égalité.

**Index** : Structure de données spéciale associée à une table de base de données pour accélérer la récupération des lignes.

**INNER JOIN** : Type de jointure qui renvoie les lignes lorsque la condition de jointure est vérifiée dans les deux tables.

**INSERT** : Commande SQL utilisée pour ajouter de nouvelles lignes de données dans une table.

**Jointure interne** : Synonyme de INNER JOIN, type de jointure qui renvoie les lignes lorsque la condition de jointure est vérifiée dans les deux tables.

**Jointures externes** : Catégorie de jointures (LEFT, RIGHT, FULL JOIN) qui incluent des lignes non correspondantes de l'une ou l'autre table.

**Jointures SQL** : Opérations permettant de combiner des lignes de deux ou plusieurs tables basées sur une colonne liée entre elles.

**LEFT JOIN** : Type de jointure qui renvoie toutes les lignes de la table de gauche et les lignes correspondantes de la table de droite.

**Lignes** : Enregistrement unique dans une table, représentant une instance de l'entité.

**Modèle entité-association (E-A)** : Outil conceptuel pour modéliser les données d'un système en identifiant les entités, leurs attributs et les relations entre elles.

**Modèle relationnel** : Façon d'organiser les données en tables composées de lignes et de colonnes, avec des relations définies entre elles.

**Modélisation relationnelle** : Processus de conception d'une base de données relationnelle, impliquant la création de tables et la définition de leurs relations.

**NOT** : Opérateur logique qui inverse une condition.

**Opérateurs logiques** : Mots-clés (AND, OR, NOT) utilisés dans la clause WHERE pour combiner ou inverser des conditions.

**Optimisation des requêtes** : Processus d'amélioration de l'efficacité des requêtes SQL pour réduire le temps d'exécution et la consommation de ressources.

**ORDER BY** : Clause SQL utilisée pour trier les résultats d'une requête selon une ou plusieurs colonnes.

**OR** : Opérateur logique qui combine deux conditions, au moins l'une d'elles devant être vraie.

**Performance** : Mesure de l'efficacité et de la rapidité d'exécution des opérations dans une base de données.

**Plans d'exécution** : Représentation détaillée des étapes qu'un SGBD va suivre pour exécuter une requête SQL.

**PostgreSQL** : Système de Gestion de Bases de Données (SGBD) relationnel open-source utilisé comme référence.

**Recherche Full-Text** : Fonctionnalité de base de données permettant de rechercher des mots ou des phrases dans des colonnes de texte de manière efficace et pertinente.

**Requêtes imbriquées** : Requêtes SQL contenant une autre requête à l'intérieur.

**Requêtes SQL** : Instructions utilisées pour communiquer avec une base de données, pour interroger, ajouter, modifier ou supprimer des données.

**RIGHT JOIN** : Type de jointure qui renvoie toutes les lignes de la table de droite et les lignes correspondantes de la table de gauche.

**Schéma E-A** : Représentation graphique ou textuelle du modèle entité-association.

**Schéma relationnel** : Représentation de la structure d'une base de données relationnelle, incluant les tables, colonnes, clés primaires et étrangères.

**SGBD** : (Système de Gestion de Bases de Données) Logiciel permettant de créer, gérer et interagir avec une base de données.

**SELECT** : Commande SQL utilisée pour récupérer des données d'une ou plusieurs tables.

**SQL** : Langage standard pour interroger et manipuler les bases de données relationnelles.

**Sous-requêtes (subqueries)** : Requête SQL qui est exécutée dans le cadre d'une autre requête, souvent un type de requête imbriquée.

**SUM** : Fonction d'agrégation qui calcule la somme des valeurs d'une colonne numérique.

**Tables** : Structures de base dans une base de données relationnelle, composées de lignes et de colonnes, stockant des données sur une entité spécifique.

**to_tsquery** : Fonction PostgreSQL qui convertit une chaîne de recherche en un type de données tsquery pour la recherche Full-Text.

**to_tsvector** : Fonction PostgreSQL qui convertit un document texte en un type de données tsvector pour la recherche Full-Text.

**Trigrammes** : Séquences de trois caractères consécutifs utilisées pour des correspondances approximatives et la recherche textuelle.

**ts_rank** : Fonction PostgreSQL qui calcule la pertinence d'un document pour une requête Full-Text donnée.

**Types d'index** : Différentes méthodes d'organisation des index, telles que B-tree et hash.

**UPDATE** : Commande SQL utilisée pour modifier des données existantes dans une table.

**WHERE** : Clause SQL utilisée pour spécifier une condition de filtre pour les résultats d'une requête.
